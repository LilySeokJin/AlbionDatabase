Mongol Khanate Logistics

We are the Logistics team of Mongol Khanate which is a primarily EU, Lymhurst based guild focused on both PvP and PvE. We offer daily activities such as Fame Farming, Ganking, Hellgates, Mage Raids, Avalonian Raids, Crystal League, and are involved in regular ZvZ Content! We are new player friendly, Territory owners with Hideouts in The Roads of Avalon and Blackzone. We are looking for newer players with 5m+ Total Fame who are willing to learn from some of our more veteran players, and dip their toes into end-game content. We are also looking for Veteran players who want to take part in End-Game Content and regular ZvZ fights.

With this content, we need to focus on management of silver, resources, items, and builds for the guild. While we manage these resources through Google Sheets, it is optimal to display information that looks aesthetically pleasing. We provide regearing for our guild for our events and because of this, it is important to make sure that we have an idea on our inventory. Introducing the Logistics website for Mongol Khanate that will implement information in this guild.

This is a private project that implements data into a Heroku database. Information is mostly stored, however, in Google Sheets where the data can be transferred directly to the website. With Heroku and Google Sheets, information is updated regularly.

Treasury Information

The Treasury page will be sync'ed with the Google Sheet which has the values for certain days and create graphs using the data. Information that will be useful would include the following:

Guild Treasury: The graph will display the amount of money in the guild with respect to time.

Donation Margins: The chart will display donations from the guild from fellow guild members.

Gain/Loss Calculation: Alongside this information, there will be an integration of many mathematical calculations.

Resources

Resources in the guild will be kept track of. These resources obtained are from gatherers from the guild. Additionally, resources will have potential calculations to create certain amounts of resources but that is still in the works. Resource calculation can be changed based on tax fees. The resources page, to reiterate, will hold these items:

Resource Management: Each resource type will be highlighted and noted.

Resource Potential: Calculations for resource crafting and fees will be taken into account.

Artifact Management: Ideally, there will be lists of artifacts stored by the guild and the information will be kept.

Database Information

The Database will have a list of every item in the game in terms of equips. Their tiers will be highlighted as well as the amount in our inventory and the storage of key items for certain builds. These builds will be displayed in filtered views of the database which are part of the builds category. The Database is comprised of the following aspects:

Master List: All the items in the database for equips. Future stretch goal includes implementing the API for the current Albion Database.

Graphic Models: Showcases relevant items to the guild and information based on what is necessary for guild contributions to meet targets for each item.

Google Sheets Implementation: Information is dragged from Google Sheets to fill the master list.

Builds

This view is a filtered view of the database which will focus on showing which builds are necessary for certain specs for each type of activity. The main meat of it is from the database from Google Sheets but using this, the builds can be shown in HTML in a more sophisticated view. This also can be used to explain why certain builds are meta. Finally, we can actually integrate a timeline as the meta changes so we can see how the meta adapts and evolves over time.

Future Stretch Goals

The website is very limited because of the time consumption to make it look extremely nice so that it will be used longterm. Because of this, we will be providing a list of possible tasks that could be done in the future:

Albion Database API: The integration of this API will allow lookup of prices for items which will allow estimates for actual worth.

Login Feature: Allowing the user to log in will make the data more private for the Logistics Team. The requirement will allow us to prevent people from other guilds to obtain information on our Logistics.

Alerts: Allowing users to have an alert feature makes it possible to recognize which resources we need.

Avalonian Roadmap: This will allow users to plot avalonian maps to different locations every day. If someone finds a path to certain areas, they can put information on here.

Calendar of Events: This idea would be extremely good if we managed to pull it off. It would integrate Discord API to generate a Calendar based on upcoming events.
